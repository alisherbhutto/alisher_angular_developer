About

Frontend engineer, banking-systems mindset

I care about the parts of an app that don't show up in a demo: validation that actually holds, error states that make sense, and forms that don't break in production.

I'm Ali Sher, an Angular Developer based in Karachi, Pakistan, currently building frontend features for Bridge — a banking repository and gateway platform used across merchant, customer, and agent workflows.

Outside my full-time role, I take on freelance frontend projects, translating designs into responsive interfaces and integrating them with real APIs for international clients.

  • LocationKarachi, Pakistan
  • FocusAngular · TypeScript · REST
  • EducationBSCS, 2019–2023
  • FreelanceFiverr, since 2023

How I work: most of my recent work sits at the intersection of Angular reactive forms, C# ASP.NET Core services, and the validation rules underneath them. When a stored procedure needs a new field, I trace it end-to-end — form control, DTO, and error message — so nothing silently breaks downstream.

I've also spent time hardening existing code: reducing complexity flagged by static analysis, fixing exception handlers that were quietly swallowing errors, and making sure the message a user sees on screen actually matches what went wrong on the backend.

I like tools that get out of the way — Git, Postman, VS Code — and interfaces that don't need a manual.
